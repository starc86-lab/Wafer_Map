## 관찰 사항

- **값이 작을수록 좋은 액션**: colormap 변경, view 토글(캐시), circle/decimals toggle — 50ms 이하 목표.
- **값이 클 수밖에 없는 액션**: 첫 Run, 보간/격자 변경, VALUE 변경 — 병렬 RBF + mesh 생성 불가피.
- **캐시 효과 확인**: view 토글의 first vs cached 격차가 크면 첫 3D 전환 비용이 큼 → prefetch 검토.

## 주의

- 각 수치는 1회 측정이라 ±20% 노이즈 포함. 안정치 필요하면 N회 평균 필요.
- OpenGL 드라이버·GPU에 따라 3D 관련 수치는 유의미하게 바뀔 수 있음.
- Windows 백그라운드 프로세스 영향 있음.
